Mixed Network Debugging

For mixed network analysis and debugging, you can use the -dms_mxnet_sql option. This option creates the mixed network database and generates a mxnet_sql.db file, which is an SQL representation of the mixed network database.

You can specify this option to debug information about connect modules, using standard SQL browsers. The information is generated at the start of the simulation. So, you can use this option instead of -ieinfo to access similar information about connect modules within the context of a mixed network. In larger designs, the -dms_mxnet_sql option is a faster and more efficient way to dump a searchable database of the connect modules and their hierarchical connectivity details in mixed-signal designs.
