Reusing Testbenches and IP

AMS/ADE enables you to reuse an existing testbench/IP in other designs. This can be done in the following three ways:

  • Advanced Testbench Reuse: With this mode, you can reuse a digital super top above the AMSD HED configuration that you are simulating in ADE. 
  • Export as IP Block:
    • Using ADE GUI: With AMS-Unified Netlister, you can export the binding information, which includes netlist, text files, and configuration, and then reuse this information later in other designs. To do so in ADE, select Tools>Export>Export as IP Block. 
    • Using the runams command: You can also export IP block using the in the runams -export command-line option.

  • Using Command-Line IP Selector (CLIPS): In CLIPS, specify a testbench and the tool shows the design hierarchy. From the hierarchy, you can switch any instance to a HED configuration. Then, the tool calls AMS-UNL to export the HED configuration as an IP and assemble the IP together with the testbench. 

    CLIPS works with XCELIUM 17.10 s007 or higher versions.

This feature is available only with XCELIUM 17.04 or higher versions.
