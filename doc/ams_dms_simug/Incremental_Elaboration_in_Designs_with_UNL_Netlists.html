Incremental Elaboration in Designs with UNL Netlists

Incremental Elaboration can be enabled in designs with UNL netlist in the primary partition and UVM or SV testbench in the incremental partition. To enable incremental elaboration, you must use the -relax_stiavm_multirun command-line option.

You must use the multi-step invocation mode, to create snapshots and run the simulation.

Single-step invocation mode is currently not supported in these designs.

To enable incremental elaboration in the multi-step mode, perform the following steps:

  1. Create the netlist and generate the ADE/UNL command-line options to build the primary partition.

  2. Build the primary snapshot.

    Ensure that you use the -relax_stiavm_multirun command-line option at this step. Also, the UNL top must be defined in the primary top.

  3. Build the incremental snapshot.
  4. Run the simulation.

For advanced testbench reuse, you must build Virtuoso IP1 and IP2 as the primary snapshots and the top-level testbench as the incremental snapshot.
