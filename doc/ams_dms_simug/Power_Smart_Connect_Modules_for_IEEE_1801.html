Power Smart Connect Modules for IEEE 1801

The Xcelium simulator mixed-signal option supports the same power-smart connect modules for the IEEE 1801 standard that are supported for CPF. The names of the power-smart connect modules have been unified to have the suffix _LPS.

The low-power intent is described in the IEEE 1801 file, and the 1801 flow is enabled by adding the -lps_1801 option to the xrun or xmelab command line. The selection and insertion of power-smart connect modules complies with the Verilog-AMS language semantics.

The power-smart connect modules parameter vpso is also supported with IEEE 1801 in the ie card configuration. This parameter is used to specify the voltage value passed to the analog block when the digital block driving it is shut off. The default value of vpso is 0.2.

In the above figure, the low-power intent for the digital block digital_A is described in IEEE 1801 format and is applied using the -lps_1801 option. In this case, the power-smart connect module can identify:

  • The power domain state of the digital_A block and use the voltage value of the supply set associated with the power domain PD1 to perform digital-to-analog transition.
  • The isolation rule applied on the digital port and propagate the correct analog value to the analog_B block.
