License Requirement Report

You can use the -licinfo command-line option with xrun command to determine the simulation-time licenses required for a specified design. The -licinfo option provides additional information about the features that use a DMSO license, which is checked out for mixed-signal design.

The -licinfo option does not check out any licenses and does not run the simulation. The software only loads the snapshot header to identify the current license requirements.

The -licinfo option generates a report on the licensing requirements using the following parameters:

  • The contents of the design (AMS, SystemC, SystemVerilog, Verilog, VHDL, and so on).
  • Other options that are specified on the command line.
  • Any arguments specified using the -uselicense Option.

The Xcelium Mixed Signal App License is needed because the following DMS technologies are used in the design:
    *  SystemVerilog Real Number Modeling(SV-RNM) Technology
    *  Advanced Mixed Signal Algorithms/Technology (Functionality, Performance or Debug)

Debugging features such as Tcl-commands force/release on wreal and save/restart require Xcelium Mixed-Signal App license; however, this cannot be determined by -licinfo.
