Connecting Design Supply Net to CPF Power Domain and Design Net

You can define a connection on mixed-signal power-supply nets based on the CPF command create_global_connection. The power net or hierarchical net is defined either in the CPF scope by using the create_power_nets command, or it is defined in the HDL scope. And, the ground connections for the power domain net or hierarchical nets can be specified in the CPF scope using the create_ground_nets command. You can associate the power supply nets to the power domain by using the CPF command update_power_domain.

To enable this feature, use the xrun command-line option, -lps_ams_connect_supply.

The following figure shows the connection between a design supply net and the CPF power domain and design net:

In a digital instance, which is under a power domain driven by design supply net, the power aware connect module inserted on the instance port is aware of the design supply net value dynamically.

To set the value of net for domain mapped nets that are also physically connected at parent level, you must use the -lps_ams_highconn_relax option. This also enables power supply net connectivity on both HDL and CPF, if supply connection exists on both.

In the following example, the tool applies the power supply connection on the top level port of this signal.

create_global_connection -net VDD -pins VDD -instances {testbench/PMUInst digInst anaInst wrvInst}
create_global_connection -net VSS -pins VSS -instances {testbench/PMUInst digInst anaInst wrvInst}
==============test.vams====================
module top(VDD, VSS);
input VDD, VSS;
electrical VDD;
wreal VSS;
dinv digInst(VDD, VSS);
ainv anaInst(VDD, VSS);
winv wrvInst(VDD, VSS);
endmodule

module testbench;
top dut(vdd, vss);
PMU PMUInst();
endmodule

module dinv (VDD, VSS);
input VDD, VSS;
endmodule

module ainv (VDD, VSS);
input VDD, VSS;
electrical VDD, VSS;
endmodule

module winv (VDD, VSS);
input VDD, VSS;
wreal VDD, VSS;
endmodule

The net testbench.PMUInst.VDD and testbench.PMUInst.VSS are connected with the port VDD and VSS of instances anaInst and wrvInst, respectively, through CPF supply nets VDD and VSS. At the same time the port VDD and VSS of the instances digInst, anaInst, and wrvInst have physical high connection in HDL source code, which introduces the multiple drivers on this net.

You can also establish a connection on mixed-signal power-supply nets, including SV-UDN nets, by using the domain mapping between macro model instantiation.

For SV-UDN power-supply nets, the UDNs connecting to the same supply net must have the same nettype. Also, for the electrical to UDN connections, you must provide the connect module specifications through ie card or connect rules.

The tool automatically creates the internal power nets if the power/ground pins of the power domains are specified using the CPF command update_power_domain, as shown below.

update_power_domain -name VREG_OUT -primary_power_net {out} -primary_ground_net {gnd}

The power net connection can be established by using the -domain_mapping command, as shown below.

set_instance top.vreg_inst25 -model VREG -domain_mapping { {VREG_OUT PD_25} }

Here is an example:

create_power_domain -name PD_25
set_macro_model VREG
    create_power_domain -name VREG_OUT
    update_power_domain -name VREG_OUT -primary_power_net {out}-primary_ground_net {gnd}
end_macro_model 

set_macro_model DECODER
    create_power_domain -name PD_DECODER
    update_power_domain -name PD_DECODER -primary_power_net {vdd} -primary_ground_net {vss}
end_macro_model 

set_instance top.vreg_inst25 -model VREG -domain_mapping { {VREG_OUT PD_25} }
set_instance top.dec_inst25 -model DECODER -domain_mapping {{PD_DECODER PD_25}}

Based on the above CPF definition, supply connection between top.vreg_inst25.out->top.dec_inst25.vdd and top.vreg_inst25.out->top.dec_inst25.vss is applied automatically.
