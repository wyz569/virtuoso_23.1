Split Objects Crossing Design Partitions Form

Specifies how to split the objects that are part of single or multiple design partitions in the top design partition. All settings implemented in this form are also applied during hierarchical editing.  

Field Description

Split Mode

Specifies the mode in which you want to split crossing objects. Valid values are: Split Objects by Type, Selected from Canvas, and Select by Net.

Select Object by Type

Splits all objects crossing single or multiple design partitions by object type.

The following options are available when Split Mode is Select Object by Type.

You can select one or more of the following objects:

  • PathSegs
  • Paths
  • Rectangles
  • Polygons

Paths with round ends or paths or path segments that can be converted to polygons are not split and are not reported as failed.

Environment Variable: autoSplitObjTypes

Split Criteria

Sets the criteria for splitting the objects.

Environment Variable: autoSplitObjCrossingPartition

Split Objects

Specifies which objects need to be split across design partitions.

  • Crossing at least one partition splits the objects that cross one or more design partitions at the area boundary of each design partition. When this option is selected, all objects that cross at least one design partition are split.
  • Crossing two or more partitions splits objects that cross multiple design partitions at the area boundary of each design partition. When this option is selected, objects crossing a single design partition are not split.

Partition Area Halo

Objects are not split if the crossing part is inside the halo of design partition area.

Override Split Crossing Objects Option

Overrides the Split Crossing Objects configuration set in the Define Design Partition form for the top design partition and subcell in which you want to split the objects.

The updates made in the Split Objects Crossing Design Partition form are not saved to the design by default. To save these changes, you should first enable Override Split Crossing Objects Option in the assistant.

Split Objects in Existing Hierarchical Partition Views

Splits the objects crossing area boundaries in the hierarchical design partition views. This option is useful if you did not enable the Split Crossing Objects option while creating the design partitions. You can also use this option when there are some new or modified objects or area boundary changes in the top design.

If you choose to split new or modified objects at hierarchical level, Cadence recommends that you first join all objects and then split them.

When this option is selected, clicking OK or Apply, displays the Split Objects Crossing Partitions form.

This option is available only when the top design partition contains hierarchical design partitions.

Selected from Canvas

Splits the crossing objects selected from the canvas.

This option is available when Split Mode is Selected from Canvas.

Select by Net

Splits the crossing nets selected from the Navigator or canvas.

The following options are available when Split Mode is Selected by Net.

Add Selected Nets

Adds the nets selected in the Navigator or canvas to the list of nets to be split across design partitions.
