Writing a Callback Function for a Form or Dialog Box

If you want your callback function to be passed the default arguments,

  • Create a callback function that takes four arguments, each of which is a symbol.
    The callback function will be passed the following arguments:

    objType

    ’form or ’dbox

    objName

    Symbol name of the form or dialog box

    objDetail

    Always nil

    moreDetail

    nil if the Help button is clicked

    Symbol name of the field if the Help or F1 key is pressed when the cursor is over a field

    ’hiFormNoField if the Help or F1 key is pressed when the cursor is not over a field

    These are the same arguments that are passed to hiHelp().

If you do not want your callback function to be passed the default arguments,

  1. Create the callback function to your own specifications.
  2. Set the value of the hiHelpAction property to a string that contains parentheses. See Example 2 below for details.

Your callback function does not have to return any specific values—hiHelp() ignores the return values from your callback.

myForm = hiCreateAppForm( ?name ’myForm ...)
putprop( ’myForm ’myHelpCB ’hiHelpAction)
procedure( myHelpCB( objType objName objDetail moreDetail "ssgg" )
     hiDisplayAppDBox( ?name ’myHelpDBox ?dboxBanner "My Help"
                       ?dboxText strcat( "My Help Text for "
                                                 get_pname( objType) ""
                                                  get_pname( objName) ) )
myForm2 = hiCreateAppForm( ?name ’myForm2 ...)
putprop( ’myForm2 "myHelpFunc( \"no help for myForm2\")" ’hiHelpAction)
procedure( myHelpFunc( helpText "t")
     hiDisplayAppDBox(?name ’myHelpDBox2 ?dboxBanner "My Other Help"
                      ?dboxText helpText)
)
