drplJitter

drplJitter(
[ ?result t_result ]
[ ?resultsDir t_resultsDir ]
[ ?freq n_freq ]
[ ?k n_k ]
[ ?unit t_unit ]
[ ?ber g_ber ]
[ ?event n_event ]
)
=> value / nil

Description

Calculates jitter from the result of pnoise analysis, where noisetype is set as pmjitter.

Arguments

?result t_result

Name of the result of pnoise analysis. By default, the name for the result of pnoise jitter analysis is pnoise_pmjitter.

?resultsDir t_resultsDir

Path to the results directory.

?freq n_freq

Frequency at which you want to calculate jitter. This value is used in case of an autonomous circuit only. Specify it as nil, in case of a driven circuit.

?k n_k

The number of cycles. The default value of k is 1.

?unit t_unit

Unit to measure jitters. Possible values are Second, UI, and ppm.

?ber n_ber

The value of BER (Bit Error Rate) when the signal level is peak-to-peak.

?event n_event

Index of an event in the result. The jitter value is calculated for each event.

Value Returned

value

Returns the jitter value.

nil

Returns nil, if the result name is incorrect and jitter cannot be calculated.

Examples

Returns a jitter value.

drplJitter(?result "pnoise_pmjitter" ?unit "Second" ?event 3 ?k 1)
