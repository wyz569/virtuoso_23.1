Troubleshoot Technology File-Related Issues in the Virtuoso Studio Design Environment

This section lists the common problems found in technology files in the Virtuoso Studio design environment and provides information about how to fix these to enable Abstract Generator to run.

Antenna Values Must Not Equal Zero

Abstract Generator checks all antenna values to ensure that their value is not 0.

Specifying Routing Grids

To enable Abstract to start, it is required to define routing grids in the technology file. If the routing grids are not defined in the technology file, the technology file will not load properly and an error message will be displayed.

Read-Only Technology File in Abstract Generator

By default, Abstract Generator creates boundary, pin, and net layer purpose pairs for the layers for which it does not find boundary, pin, and net purposes in the technology file, and then tries to save these layer purpose pairs in the technology file. Because the technology file is read-only, the layer purpose pairs are not saved and a warning message is displayed.

To enable Abstract Generator to work with the read-only technology file, the ReadOnlyTechnology option has been added to Abstract Generator. When this option is set to TRUE, Abstract Generator does not try to save to the technology file the layer purpose pairs that it creates.

You can set the ReadOnlyTechnology option to TRUE as follows:

absSetOption(“ReadOnlyTechnology” “true”)

Layer Not Fully Defined

Abstract Generator requires that all the layers used in the process are fully defined in the appropriate technology file sections. Abstract Generator issues the message ABS-217 if a layer is defined in the LEFDefaultRouteSpec (or the specified constraint group) but is not included in the functions subsection in the layerRules section or if the layer function in the functions subsection is not specified as metal.

Related error message: ABS-10016
