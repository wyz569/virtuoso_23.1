Abstract Generator Log, Replay, and Record Files

During an abstract session, the abstract.log, abstract.record, and abstract.replay files are created. In all cases, these files will have the same numbering to identify the correct abstract shell.

File Name Description

abstract.log file

Stores all the information commands and the info, warning, and error messages issued by Abstract Generator during the current session. Instead of using the default abstract.log file, you can use the -log option to specify a different log file name.

Example

To create a log file named session.log, specify the following:

% abstract -log session.log

If there are multiple active abstract sessions in the same test case directory, then separate log files with names such as abstract.log, abstract.log.1, and abstract.log.2 are created.

abstract.log.cdslck file

Along with a log file, an abstract.log.cdslck file is also created for each new abstract session. An abstract.log.cdslck file indicates that an abstract session is currently running, for which the log file is being used. The abstract.log.cdslck file exists only for the current abstract session, and is deleted when you exit the session. The abstract.log is overwritten if there is no abstract.log.cdslck file present in the current directory. If you specify a different log name for a session, then the corresponding <user_defined>.log.cdslck is created.

Example

For session1.log file, a corresponding session1.log.cdslck file is created. If a session1.log.cdslck already exists, and you invokes an abstract session by specifying: 

% abstract –log session1.log

In such cases, an error is displayed and Abstract Generator is not invoked. To invoke Abstract Generator, you either need to specify a different log file name or close the currently active session and try again.

abstract.record

Every command and option you use during the current abstract session is recorded in a .record file. The numbering of the abstract.record and abstract.log files is always in sync.

abstract.replay

For every abstract session, an abstract.record file is created. You can use the abstract.record file from a previous run as the replay file for the current session by specifying: 

% abstract –replay abstract.record

To save your previous work and to avoid the abstract.record file from being overwritten, the abstract.record file is renamed as abstract.replay, and then it is replayed. A new abstract.record file is created for the current session.
