Technology File Requirements for Abstract Generation

This section describes how Abstract Generator handles technology information in the Virtuoso Studio design environment, outlines the technology information required to run Abstract Generator on OpenAccess, identifies potential technology file problems, and suggests possible solutions to those problems.

You can generate technology information for Abstract Generator, supplement that information by importing LEF, and modify technology information using the Technology File Manager.

Technology information provides details of the process technology used during IC fabrication, including the names of layers, colors and fill patterns, GDSII layer mapping data, and design rules for various layers and vias. Abstract Generator can handle OpenAccess technology information associated with your design library.

You can list the technology information required to run Abstract Generator on OpenAccess. This includes the definition of layers and layer rules, layer functions, physical rules, and vias along with specific requirements for antenna calculation, and viewing data in the Virtuoso Layout Editor.

The following are the three main sources of technology information for a design library in the Virtuoso Studio design environment:

  • A binary technology file, called tech.db, stored in the library directory
  • A separate technology library, which is attached as a property to the design library
  • A display resource file, display.drf, stored in the library directory
