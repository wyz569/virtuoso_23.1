Import Layout Form

Use the Import Layout form to create cellviews from existing layout information in GDSII format.

Field Description

GDSII Filenames

Lets you specify the names of the GDSII files that you want to import.

Map GDSII names

Lets you specify whether you want GDSII names mapped to upper case, lower case, or to not be mapped at all.

Layer Map Table

Lets you specify the name of the layer map file you want to import.

Default Bin

Lets you select the bin into which the cells are imported. This option applies only to any new library cells being imported. Cells imported previously are not moved.

During import, Abstract Generator creates layout views for the cells in the library. Some types of cells, such as sites and generated vias, do not need to be processed, so Abstract Generator moves them directly into the Ignore bin. A cell is placed in the Default Bin only if it meets the criteria below.

  • There is a valid layout view.
  • The cell is not a via.
  • The cell is not a site

Template Filename

Lets you specify a stream template file to be imported into Abstract Generator.

If you do not select a template file, Abstract Generator creates a default file called PcsStreamIn.template in the .abstract/import directory. This file has default option settings of No Mapping and Core bin and can be used to create your own template file.

Regardless of whether a template is supplied or not, the values of the GDSII Filenames and the Map GDSII names options are always the values specified in the form. See Chapter 2 of the Design Data Translator’s Reference.

The subsection techControl, added to the technology file for ITDB (Incremental Technology Database) support, necessitates the specification of a set of ordered reference libraries to allow inheritance from these libraries.

Technology Reference Library Name

Allows you to specify one or more valid reference technology libraries. The specified referenced technology libraries must be present in the cds.lib file and must be readable.
