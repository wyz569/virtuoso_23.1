Exporting LEF in Abstract Generator

The Export LEF form generates LEF descriptions for cells in one or more bins.

  1. Choose File – Export – LEF.
    It displays the Export LEF form.
  2. In LEF Filename, specify a path and name for the LEF file to export or use the Browse button to specify a location.
  3. Select Export Geometry LEF Data and Export Tech LEF Data to choose the combination of geometry LEF and technology LEF to be exported. All the LEF information is written to the specified LEF file name.
  4. Select Cell List Filename to export LEF information for only the specified cells. The list of cells, whose LEF information is to be exported, is specified in a cell list file, with one cell name per line. Specify the name of this cell list file in the field or use Browse to specify the file’s location. This is an optional field.
    If you specify a cell list file, the Export LEF for Bin option gets disabled.
  5. In LEF Version, select the version of LEF that you want to export.
  6. In Export LEF for Bin, select the bins for which you want to export the LEF data. You can export LEF for all the cells in your library or on a bin-by-bin basis.
  7. In Bus characters and Divider character, select the characters you want to use to represent buses (< > or [ ]) and pseudohierarchy ( / or | ), respectively, in your LEF.
  8. Click OK to export the LEF files.

The resultant LEF file can be imported into the target P&R system.

Including DATABASE MICRONS in the Exported LEF

To set DATABASE MICRONS values in the LEF information exported from the CDB Abstract Generator, you must set the gec3DBUPerUU value in your CDB technology file.

In OpenAccess, the DATABASE MICRONS value is taken directly from the DBUPerUU value defined in the technology file.
