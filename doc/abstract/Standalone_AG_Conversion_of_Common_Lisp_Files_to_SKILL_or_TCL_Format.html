Conversion of Common Lisp Files to SKILL or TCL Format

This section describes how to convert any Abstract Generator replay files you have from common-lisp format into SKILL or TCL format. As SKILL is used extensively in the Virtuoso Studio design environment, Cadence recommends that you convert any replay files from common-lisp to SKILL format. To do this, use the Abstract Common Lisp Converter, absclconv.

You should use absclconv to convert only Abstract Generator common-lisp files. It is not designed as a generic common-lisp conversion utility.

The command accepts the following arguments.

absclconv
{ -tcl | -skill }
clFile
newFile

These arguments are described in detail below.

-tcl | -skill

Specifies the format of the file to be generated from the common-lisp file.

clFile

Specifies the common-lisp file to be converted.

newFile

Specifies the name of the file to be created in the new format.

For ease of use, the following shortcuts are provided.

  • To convert a common-lisp file directly to SKILL format, use the abscl2il link.
    abscl2il clFile ilFile
    This creates a new SKILL file, ilFile, based on the common-lisp file clFile.
  • To convert a common-lisp file directly to Tcl format, use the abscl2tcl link.
    abscl2tcl clFile tclFile
    This creates a new Tcl file, tclFile, based on the common-lisp file clFile.

In all cases, if a file with the same name already exists, it is overwritten.

absclconv preserves comments and blank lines, so a command that is commented out in the common-lisp version remains commented-out in the SKILL and Tcl versions. (The comments are displayed differently in SKILL and Tcl.)

The example below shows how a common-lisp command converts to Tcl and SKILL formats.

Common-Lisp Tcl SKILL

(abs:Select “param1” “param2”)

absSelect “param1” “param2”

absSelect (“param1” “param2”)
